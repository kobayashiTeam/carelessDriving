技術要素

(1)敵の移動は一定間隔ごとにWayPoint領域に導くことで柔軟性のあるAIを実現

<br>

工夫

(1)Waypointを「点」ではなく内外2層の円形領域として扱い、

外周接触時は減速/内周侵入時は到達判定

を行うことで、急旋回によるコースアウトを抑制。

また、旋回中は加速を停止し、慣性による横滑りを防止した。

これにより、機械的な折れ線移動ではなく、カーブを安全に曲がる自然な走行挙動を実現。

<br>

(2)応援NPC（FBXモデル）をコース上に多数配置。

ゲーム開始時に全NPCを同時生成すると描画負荷によるカクつきが発生したため、

数秒間に分散して生成/不要エリア通過後は破棄

を行い、メモリ使用量と瞬間負荷を削減。

これにより演出密度を維持したまま、ゲーム開始時のフレーム落ちを軽減。

<br>

(3)特定NPC付近でプレイヤーカメラが対象を注視する演出を実装。

カメラを瞬時に切り替えるのではなくLerp補間で徐々に回転させることで、不自然な視点変化を防止。

また、Y軸回転角に制限を設け、

真後ろを向かない/プレイ視認性を維持するよう調整した。

これにより演出性を維持しつつ、レースゲームとしての操作性を損なわない視線誘導を実現。

<br>

(4)居眠り中に視界が暗転するギミックを実装。

単純な画面暗転ではなく、

Skybox切替/Emissionによるヒント強調/ライトマップ差し替え

を組み合わせて昼夜変化を表現。

また、ライトマップ切替時の一時的な描画乱れを隠すため、暗転演出中に処理を実行した。

これによりゲームプレイを維持したまま、自然な視界喪失演出を実現。

<br>

仕様技術/環境

Unity 600.2.6f2/C#/URP
